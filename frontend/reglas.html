Manual del Usuario y Reglas Léxicas

1. Estructura Básica de un Programa

Todo bloque de reglas en PayLogic debe estar contenido obligatoriamente dentro de las palabras clave principales que marcan el inicio y fin del flujo de ejecución.

INICIO
  // Aquí van tus reglas, cálculos y variables
FIN

2. Reglas de Escritura (Tokens)

  • Identificadores (Variables): Deben iniciar con una letra (mayúscula o minúscula) o un guion bajo (_). Pueden contener números después de la primera letra. Ej: bono_ventas, tasa_retencion.
  • Números: Se aceptan valores enteros o decimales. No deben llevar comas separadoras de miles. Ej: 1500, 250.50.
  • Asignación de Valores: Para darle un valor a una variable y actualizar su valor progresivo, obligatoriamente se usa el operador :=.
  • Fin de Instrucción: Toda línea que ejecute una acción o declare una variable debe terminar obligatoriamente con un punto y coma ;.

3. Palabras Reservadas

INICIO FIN VAR CONST SI ENTONCES SINO MIENTRAS RETORNAR SISTEMA

4. Operadores Permitidos

Aritméticos:

+ - * /

Relacionales (Validación de límites):

> < >= <= == !=

5. Ejemplo de Generación de Resultados

INICIO
  VAR ventas_totales := 5000;
  VAR bono := 0;

  SI ventas_totales >= 3000 ENTONCES
    bono := ventas_totales * 0.10;
  SINO
    bono := ventas_totales * 0.05;

  RETORNAR bono;
FIN
